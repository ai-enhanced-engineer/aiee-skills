# Angular Tooling Examples

Practical workflows for AI-assisted Angular development.

## New Project Setup with AI

```bash
# 1. Create new project with Claude config
ng new my-dashboard --ai-config=claude --standalone --ssr

# 2. Add MCP server to Claude Code
claude mcp add angular-cli --scope project -- npx -y @angular/cli mcp

# 3. Verify MCP connection
claude mcp list

# 4. Install component library (PrimeNG recommended)
npm install primeng primeicons
npm install @primeng/themes

# 5. Configure PrimeNG in app.config.ts
```

## Dashboard Generation Workflow

```
Prompt: Generate TypeScript interfaces for a B2B dashboard with:
- DashboardMetrics (revenue, users, conversion, trend)

- Conversation (id, customerId, messages, status)
```

```
Prompt: Create an Angular 21 standalone dashboard layout with:
- Sidebar navigation (collapsible)
- Header with user menu
- Main content area with router-outlet
```

### Step 3: Generate Feature Components

```
Prompt: Generate a MetricsWidget component that:
- Accepts DashboardMetrics via input()
- Displays 4 metric cards in responsive grid
- Uses computed() for formatting
- Has loading and error states
- Includes ARIA labels
- Uses @if/@for control flow
```

### Migration Steps

```typescript
// BEFORE: Legacy component
@Component({
  selector: 'app-user-list',
  template: `
    <div *ngIf="loading">Loading...</div>
    <ul *ngIf="!loading">
      <li *ngFor="let user of users">{{ user.name }}</li>
    </ul>
  `
})
export class UserListComponent {
  @Input() users: User[] = [];
  loading = false;
}

// AFTER: Modern Angular 21
@Component({
  selector: 'app-user-list',
  standalone: true,
  template: `
    @if (loading()) {
      <div role="status" aria-live="polite">Loading...</div>
    } @else {
      <ul>
        @for (user of users(); track user.id) {
          <li>{{ user.name }}</li>
        } @empty {
          <li>No users found</li>
        }
      </ul>
    }
  `
})
export class UserListComponent {
  users = input<User[]>([]);
  loading = input(false);
}
```

## CI/CD Integration

### GitHub Actions Workflow

```yaml
name: Angular CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Test
        run: npm run test:ci

      - name: Build
        run: npm run build -- --configuration production

      - name: Score AI-generated code
        run: |
          npm install -g @anthropic-ai/web-codegen-scorer
          web-codegen-scorer score ./src/app --threshold 75

      - name: Upload build artifacts
        uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/
```

### Pre-commit Hook

```bash
#!/bin/bash
# .husky/pre-commit

# Run linter
npm run lint || exit 1

# Score changed Angular files
CHANGED_FILES=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.ts$')
if [ -n "$CHANGED_FILES" ]; then
  web-codegen-scorer score $CHANGED_FILES --threshold 70
fi
```

## IDE Configuration Files

### .claude/instructions.md (Generated by ng)

```markdown
# Angular Development Instructions

## Framework Version
Angular 21+ with zoneless change detection

## Code Patterns
- Use standalone components exclusively
- Use signals for reactive state (signal, computed, effect)
- Use input()/output() for component I/O
- Use @if, @for, @switch control flow
- Use inject() for dependency injection

## Accessibility
- All interactive elements must have ARIA labels
- Color contrast ratio >= 4.5:1
- Keyboard navigation required
- Focus management for modals/dialogs

## Testing
- Unit tests with Jasmine/Jest
- E2E tests with Playwright
- Minimum 80% code coverage
```
